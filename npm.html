npm命令

  • git clone git@github.com:Sunny-zz/1709.git

  • https://github.com/Sunny-zz/1709.git

  • github 分支 gh-pages xxx.github.io/app1

  • master 主分支
  • index.html style.css公共
  • about.html help.html
  • ls -a 查看隐藏文件
  • .git 说明本地文件夹是一个github仓库
  • git branch --查看当前仓库的分支 *说明 当前位于某个分支 -r 查看远端的分支
  • git brandch 分支名 --创建新的分支
  • git branch gh-pages 当创建新的分支的时候分支里面的内容默认和master分支一样
  • git checkout 分支名 --切换分支
  • git checkout -b [yourbranch] 创建新的分支并切换过去
  1. 克隆到本地 git clone
  2. 新建分支 gh-pages 并切换git
  3. 分支上新建 index.html 并上传 git add . git commit -m'11' git push
  4. 访问 git用户名.github.io/hellogithub 代码托管工具本地 网上
  5. 网上有仓库并且仓库内有内容 仓库克隆到本地 git clone 地址(http ssh)
    • git add . git commit -m'xx' git push git pull
  6. 把本地的项目上传到网上的空仓库git init git add . git commit -m'xx' git push
    • ssh-keygen 主目录下会生成 .ssh 文件 内部会生成 两个文件 其中一个就是公钥 id_rsa.pub 分支
  7. git用户名.github.io 这个仓库 下的 index.html 能直接访问用户名.github.io
  8. 如果一个仓库(除了 xxx.github.io 之外)内有 gh-pages 分支下的 index.html 能直接访问用户名.github.io/仓库名node npm
  9. apm install emmet atom编辑器安装插件sudo apt-get install curl curl --versionnvm node version manger
  10. 使用nvm可以安装任何版本的node
  11. sudo apt-get install curl
  12. curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.29.0/install.sh | bash
  13. nvm ls-remote 查看nvm可以安装哪些node版本npm -v
  14. nvm use stable
  15. npm node package manager新建一个文件夹npm-demonpm init
  16. npm i jquery@x.x.x --save
  17. npm uninstall jquery
  18. npm i xx --save-dev
  1. 安装工具包压缩js html css 新的js语法自动编译为es5语法
  2. .gitignore 文件 git上传时 写在该文件内的文件被忽略上传将本地仓库删除
  3. 克隆网上的仓库到本地 将node_modules删除再上传
  4. 此时网上就没有了 node_modules
  5. 然后本地 npm i node_modules 就会出现
  6. 此时 新建 .gitignore 再上传 此时本地的node_modules 就会上传不上去了
